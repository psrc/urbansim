Creating a Developer Setup

This section provides instructions for configuring your computer to do software development on one or more Opus packages.

Choosing a Development Environment

In general, if you are doing much software development, you will benefit a lot from using an integrated development environment (IDE) that supports for the range of activities involved in, such as such as editing code, running tests, debugging code, connecting to a code repository, etc.  There are numerous such IDEs for Python, and unfortunately no one IDE fits all needs.  Within CUSPA, we tend to use one of three editors for Python, and one of three editors for LaTeX files.

Configure your Python editor appropriately.  The software developers in CUSPA typically use Eclipse for talking to CVS, Wing for debugging, and either or both for editing Python code.  Here are instructions for configuring Wing and configuring Eclipse.

If you need to edit the Opus and UrbanSim Reference Manual and Users Guide (as opposed to just reading it), you’ll need to use LaTeX since our documentation is written in LaTeX.  Here are instructions for configuring a installing and configuring a LATEX document preparation system.

Getting the Code for the CUSPA Opus Packages

As a developer, you also may need to modify code in one of the Opus packages in the base CUSPA distribution (e.g. in the installer).  If this is the case, you need to get that code and configure your computer appropriately so that Python will find the files you want:

  1. Contact CUSPA to get CVS access to the Opus repositories.
  2. Download the desired Opus packages from the CUSPA CVS repository and put them into your workspace directory.
  3. Make sure that your workspace directory is in your PYTHONPATH.  This will ensure that Python uses these packages instead of the same-named Opus packages installed in its 'site-packages' directory.
  4. (optional) If you wish, you may uninstall the Opus located in Python's 'site-packages' directory, however this is not strictly necessary since Python will use the Opus packages it finds in your PYTHONPATH before it uses the ones installed in 'site-packages'.

You now should have a directory structure similar to the following: 

C:\workspace
    opus_core/                <-- if this is one of the packages you got from CVS
    urbansim/                 <-- if this is one of the packages you got from CVS
    ...
    atlantis/
        docs/
        ...
        __init__.py
        opus_package_info.py

Now you can make changes to both the atlantis and other Opus packages in your workspace, and use CVS to keep track of the changes between your copies of the Opus packages and those in the CUSPA repository, update your copies with changes from the repository, etc.

Note that the code in the CVS repositories is updated frequently (often multiple times per day). If you do not frequently update your code from the repositories, it will become very difficult to keep your changes in synch with the code in the repository. Be warned!

If you make a change that you believe merits inclusion in the CUSPA repository, please let us know so that we may incorporate that change into the repository. The more we all improve the code, the better for all of us.

In general, it is a bad idea to directly edit code to Python’s ‘site-packages’ directory (the default location used for most installers, including our installer). These changes would be over-written when you upgrade the associated Python package, e.g. when you upgrade Opus. If you need to modify such code, follow the directions in this section.
