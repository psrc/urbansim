Testing Your Install

There are two steps to testing your install:

  1. Test that Python can find your Opus packages.
  2. Test that the Opus packages pass their unit tests.

Can Python find the Opus packages?

Open a command window. 

  • On Windows, this can be done using the Start menu — choose “Programs,” then “Accessories”, then “Command Prompt.” 
  • On Linux, this is the shell. 
  • On the Macintosh, choose the “Terminal” application under “Applications/Utilities.” 

Now check that Python itself is installed and working by typing python at the command prompt to start a Python session:

C:\>python
Python 2.4.3 - Enthought Edition 1.0.0.beta4 (#69, Jul 13 2006, 15:35:31) [MSC v.1310 32 bit (Intel)] on win32
Type "help", "copyright", "credits" or "license" for more information.
>>>

Once you’re in Python, type in a simple expression and see if you get a reasonable answer: 

>>> 3+4 
7 
>>>

Now test that the opus_core and urbansim packages are correctly installed by checking that you can get the version info for them:

>>> from opus_core.opus_package_info import package as opus_core_package
>>> opus_core_package().info()
Opus package opus_core
opus_core, version 1.0.beta2
Required Opus packages for opus_core: []
Required external packages for opus_core: ['numpy>=1.3.2', 'MySQL-python>=1.2.0']

>>> from urbansim.opus_package_info import package as urbansim_package
>>> urbansim_package().info()
Opus package urbansim
urbansim, version 1.0.beta2
Required Opus packages for urbansim: ['opus_core']
Required external packages for urbansim: []
>>>

If you have already created your new Opus package it also should work (replace 'atlantis' with your package's name):

>>> from atlantis.opus_package_info import package as atlantis_package
>>> atlantis_package().info()
Opus package atlantis
atlantis, version 0.0-1
Required Opus packages for atlantis: ['opus_core']
Required external packages for atlantis: ['numpy>=1.3.2', 'MySQL-python>=1.2.0']
>>>

Do the unit tests pass?

As an optional part of testing the installation, all of the base Opus packages have unit tests that should succeed.

The unit tests in each Opus package are set up to run as standalone Python programs: simply invoke Python on the file containing the unit test. For instance, to test the Python module opus_core.tests.test_sampling_toolbox.py installed in ‘site-packages’ on Windows, open a command shell and type:

python C:\Python23\Lib\site-packages\opus_core\tests\test_sampling_toolbox.py
..
----------------------------------------------------------------------
Ran 2 tests in 0.000s

OK

C:\Documents and Settings\socha>

where a period gets printed each time a test is run, followed by a summary at the end.  If you have installed Opus to a different directory, use that path in the command, above.

For Linux or the Macintosh, the file ‘test_sampling_toolbox.py’ will be in the directory structure you unzipped from the ‘.tar.gz’ file used to build Opus and UrbanSim, if you haven’t deleted it already—in that case you can also just navigate to that directory and type python test_sampling_toolbox.py.

To run all of the unit tests defined in the opus_core package, and thus perform a more comprehensive test of your installation, invoke opus_core’s all_tests.py über-test module:

python C:\Python23\Lib\site-packages\opus_core\tests\all_tests.py 
(Note that the C:\Python23 directory above might be different on your machine)

(Caution: particularly if you don’t have a fast machine, running all_tests.py may be time consuming, but you can always abort the tests.)

Every Opus package has a ‘tests’ directory with a ‘all_tests.py’ module. The |all_tests| module will scan all files in the package searching for unit tests. It will run all that it finds. Note: unit tests that are ‘hidden’ behind an if __name__ == '__main__': check will not be found.

As part of our automated build process at CUSPA, we automatically run ‘all_tests’ every time a new version of the code is checked into our cvs repository.
