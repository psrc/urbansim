Installing Opus & UrbanSim

This section provides instructions for installing Opus, UrbanSim, and the external software required by Opus and UrbanSim. 

Opus and UrbanSim are intended to be platform-independent. We have successfully installed and used the system under the Windows, Linux, and Macintosh operating systems. These installation directions have been tested with Windows XP, SuSe Linux, and Macintosh OS X Version 10.4.5 with both PPC and Intel processors.

Opus and UrbanSim rely on numerous other open-source systems, in particular MySQL, Python, Traits and a non-trivial set of Python packages. Fortunately, on Windows, the Enthought Python Edition for Windows installs both Python and all of the Python packages needed by Opus and UrbanSim.  For Linux and Macintosh, however , there is not (yet) such a bundled set of packages, so the process of installing all the supporting software on those systems can be a bit tedious, although hopefully straightforward.

Opus and UrbanSim work with Python 2.4.3 or 2.3.5, and with MySQL 5.0 or 4.1.18.  In general, we recommend you use the most recent snapshot of these external applications, though if you are have problems, try using one of the versions mentioned here.  These instructions assume you are using Python 2.4.3 and MySQL 5.0.  If you are using different versions, adjust the instructions accordingly.

To help illustrate how to setup UrbanSim for use by your own agency, these installation directions sometimes refer to a fictitious Atlantis Metropolitan Planning Organization (MPO). Where you see the word “atlantis” you may safely substitute the appropriate name for your application.

Check the release notes for significant code or data changes.  If you have your own Opus package or an existing baseyear database you may need to modify your code and data.

Doing all but the last step creates a “Modeler Setup,” in which the base Opus packages are installed in Python’s ‘site-packages’ directory. This is the most common case. The directory structure will be similar to the following, where atlantis is your own Opus package:

C:\Python24\Lib\site-packages
    opus_core/
    urbansim/
    ... (more Opus packages) ...

C:\workspace
    atlantis/
        docs/
        ... (more directories and files) ...
        __init__.py
        opus_package_info.py

C:\Python24\Lib\site-packages is the location of Python site-packages on your computer. (For Linux or Macintosh, this line would be different, but would still be the directory in which Python site-packages are installed.  Similarly, C:\workspace would be a subdirectory of your home user directory.)

Doing the last optional step in the installation steps listed above produces a “Developer Setup,” with the base Opus packages in your own workspace. This less common case allows you to edit the base Opus packages. We suggest you only do this if you know you will be changing code in the base Opus packages. (It’s not hard to change later if you need to.) This setup produces a directory structure similar to:

C:\workspace
    atlantis/
        docs/
        ... (more directories and files) ...
        __init__.py
        opus_package_info.p
    opus_core/
    urbansim/
    ... (more Opus packages) ...

Uninstalling Opus

If you no longer wish to have Opus on your computer, you can uninstall it via the normal Python mechanisms. If you used the Windows installer, you can uninstall Opus via the “Add or Remove Programs” application in your Control Panel; look for a name such as “Python 2.3 opus-0.8”. If you installed it by unzipping it to a particular directory, just delete that directory.
