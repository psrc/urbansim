Using LaTeX with Opus

This section provides instructions for using LaTeX.  The Opus and UrbanSim Reference Manual and Users Guide (aka Manual) are written using the LaTeX document preparation system.  Advantages of LaTeX are that it is open-source, platform independent, provides a rich set of features for indexing and cross-referencing, and is the standard used for documenting Python packages. It also typesets elegantly, particularly for mathematical formulae. The disadvantage is that it is not a WYSIWYG (“what you see is what you get”) system.  Instead, you edit a source document with a text editor, then use LaTeX to format it and produce a pdf file and / or a set of html pages.  If you are new with LaTeX, take a look at the Getting Started with TeX, LaTeX, and Friends page.

The source for the Manual resides in the opus_docs Opus package in the CUSPA subversion repository.  The root LaTeX source file is 'userguide.tex'.  It includes a set of other .tex files, normally one file per chapter.  The latex files use the Python latex documentation standards and latex definition files (although not yet consistently); see http://www.python.org/doc/current/doc/doc.html. 

Installing a LaTeX environment

There are many applications and environments that support editing LaTeX files.  

On Windows, folk in our group at the UW tend to use the Texlipse Eclipse plug-in, that provides decent LaTeX support, or the Miktex version of LaTeX.  You might want to try the proTeXt installation that is based on the MiKTeX distribution, and includes some additional components including the TeXnicCenter LaTeX editor.  SuSe comes with LaTeX, as do most other Linux distributions.

On the Mac, closet UNIX hackers will probably prefer to interact with LaTeX via the command line.  If you’re comfortable with that, use the Fink distribution of LaTeX. To install LaTeX this way, after you’ve got Fink installed (Section 3.2.1), use the FinkCommander graphical interface to install the tetex-base package.  You just need tetex-base, not the whole system. Even so, this will install an alarmingly large number of megabytes of stuff on your system - but it should all go smoothly.  You can now interact with latex and pdflatex from the command line in the usual way.  If the phrase “interact with latex and pdflatex from the command line in the usual way” sounds like a lot of new stuff to learn, as opposed to having the familiarity of an old shoe, you might want to check out one of the Mac ports of LaTeX that provides a more Mac-like graphical interface instead.  See http://www.esm.psu.edu/mac-tex/ or http://www.macdevcenter.com/pub/a/mac/2004/02/03/latex.html for more information.  Or use the Texlipse Eclipse plug-in.

Running LaTeX

The easiest way to rebuild a local version of the pdf and html documentation for the Manual is to use a Python script that is in the opus_docs Opus package:

  1. Get the opus_docs package from the CUSPA subversion repository and put it in your workspace (e.g., to C:\myworkspace)
  2. Open a command window and navigate to the location of your opus_docs/docs/latex directory (e.g. to C:\myworkspace\opus_docs\docs\latex)
  3. Run the commands to build a local copy of the documentation:
python builddocs.py

The new pdf file, and associated html pages, will be in the opus_docs/docs/latex directory

Alternatively, you can issue the individual commands:

  1. Get the opus_docs package from the CUSPA subversion repository and put it in your workspace (e.g., to C:\myworkspace)
  2. Open a command window and navigate to the location of your opus_docs/docs/latex directory (e.g. to C:\myworkspace\opus_docs\docs\latex)
  3. Run latex on the file ‘userguide.tex’ (the root LaTeX source file) to produce the pdf file:
pdflatex userguide

If you get an error message that the file ‘userguide.ind’ wasn’t found (this will happen the first time you do the above command), just hit return and continue, since the next step will create this index file.

  1. Generate the index:
makeindex userguide
  1. Run latex again to incorporate the index and resolve any cross-references:
pdflatex userguide

The new pdf file will be in this directory.

  1. (optional) To create the html version of the manual, run the latex2html command:
latex2html -local_icons -bottom_navigation -address "info (at) urbansim.org" userguide
