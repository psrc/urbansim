Installing Opus and UrbanSim using an IDE (Interactive Development Environment)

Choosing a Development Environment

In general, if you are doing much software development, you will benefit a lot from using an integrated development environment (IDE) that supports for the range of activities involved in, such as such as editing code, running tests, debugging code, connecting to a code repository, etc.  There are numerous such IDEs for Python, and unfortunately no one IDE fits all needs.  Within CUSPA, we tend to use one of three editors for Python, and one of three editors for LaTeX files.

Configure your Python editor appropriately.  The software developers in CUSPA typically use Eclipse for talking to subversion, Wing for debugging, and either or both for editing Python code.  Here are instructions for configuring Wing and configuring Eclipse.

If you need to edit the Opus and UrbanSim Reference Manual and Users Guide (as opposed to just reading it), you’ll need to use LaTeX since our documentation is written in LaTeX.  Here are instructions for configuring a installing and configuring a LATEX document preparation system.

Getting the Code for the Opus Packages

  1. Download the desired Opus packages from the CUSPA subversion repository at https://trondheim.cs.washington.edu/svn/opus and put them into your workspace directory.
  2. Make sure that your workspace directory is in your PYTHONPATH.

You now should have a directory structure similar to the following: 

C:\opusworkspace
 opus_core/ <-- if this is one of the packages you got from subversion
 urbansim/ <-- another package from subversion
 ...
 atlantis/
 docs/
 ...
 __init__.py
 opus_package_info.py

Now you can make changes to the atlantis and other Opus packages in your workspace, and use subversion to keep track of the changes between your copies of the Opus packages and those in the CUSPA repository, update your copies with changes from the repository, etc.

Note that the code in the suversion repository is updated frequently (often multiple times per day). If you do not frequently update your code from the repository, it will become very difficult to keep your changes in synch with the code in the repository. Be warned!

If you make a change that you believe merits inclusion in the CUSPA repository, please let us know so that we may incorporate that change into the repository. The more we all improve the code, the better for all of us.
