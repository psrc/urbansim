Installing Opus & UrbanSim on Macintosh OS X

This following instructions are for installing Opus & UrbanSim on Macintosh OS X.  These installation directions have been tested with Macintosh OS X Tiger and Leopard with Intel processors.

Step 1: Installing Python 

Opus and UrbanSim are implemented in Python, making heavy use of numerical Python packages and other software. For the Mac we recommend using Python version 2.5.1. Mac OS X Leopard (10.5) comes with Python 2.5.1 already installed, so you don't need to update your version of Python. If you are running an earlier version of the OS (10.4.9 or earlier), the default version will be 2.3.5 or earlier and you will need to update to Python 2.5.1.

The official Python website includes documentation, downloads, and much else. Go to the Python download tab and get "Python 2.5.1 for Macintosh OS X". This is an installer that works on both PPC and Intel Macs. Double-click on the image to mount it, then double-click on the MacPython.mpkg package icon to install. When you're done you can unmount the image by clicking on the triangle symbol next to it in the Finder.

A note on Python versions: Tiger comes with Python version 2.3.5 installed.  However, the universal installers for the required packages are only available for newer versions of Python, so you need a newer version. (We  suggest leaving Apple's installed version alone - there is no need to uninstall it. When you install Python 2.5 the installer will put it earlier on the search path so that it will be found instead. You can still get to the old version via /usr/bin/python.)

(optional) Step 2: Installing MySQL

MySQL is an efficient, open-source SQL database. You don't need it to run the basic Eugene tutorial, although for serious use with your own dataset you will want a database of some sort. (We are steadily reducing the dependencies on the particular database used, however, with the goal of allowing Opus and UrbanSim to be used with any SQL database.) If you use MySQL you can either have a local copy or run on a shared server.

If you don't install MySQL right away, you can return and install it later if you discover you need it.

Opus works with MySQL 4.1 and 5.0.

  1. Choose which version of MySQL to use, and download the binary installer for your version of the OS and processor type from http://dev.mysql.com/downloads/.
  2. Install and configure MySQL.

In addition to MySQL itself, you might get the MySQL Query Browser (for prowling around in UrbanSim databases using a graphical interface), and the MySQL Administrator. These are both available as binaries.

See the Opus/UrbanSim MySQL Installation and Configuration Page for additional specific configuration steps.  This page is written for Windows, but is straightforward to use for the Mac as well.

Step 3: Installing Python Libraries

Unfortunately, there is no download for the Mac that contains all of the Python libraries needed by Opus and UrbanSim, so several steps are needed to install the Python libraries. In most cases, obtaining these packages is made simpler by the use of Easy Install, a python module that can help you download and build python packages. For more on Easy Install, visit the PEAK webpage.

Download ez_setup.py to your computer. Run it (from the directory it's in) by typing 

$ python ez_setup.py
We will use this later to install a number of packages.

Now you can start downloading python packages.

If you are running a version of OS X earlier than 10.5, such as 10.4.9, download and install the following:

  • wxPython 2.6.3.3 from sourceforge. Later versions of wxPython will not work.
  • pytz from the downloads page on pythonmac.org. This is also not necessary on 10.5/Leopard.
  • python-dateutil, also from pythonmac.org.

If you are running 10.5/Leopard, the three packages above are not necessary.

Now download and install the Superpack from http://trichech.us/?page_id=5. This includes the packages NumPy 1.0.4, SciPi 0.7 and mathplotlib 0.90.1, all of which are necessary.

Install the following packages using Easy Install:
  • Traits: Go to 
    http://code.enthought.com/enstaller/eggs/macosx/
    and navigate to the correct package, which will have a name like 
    enthought.traits-2.0.1b1-py2.5-macosx-10.3-fat.egg
    or something similar. Download it, and then use 
    $ easy_install [filename]
    to install it.
  • SQLAlchemy 0.4: Download the tarball (currently for SQLAlchehmy 0.4 from http://www.sqlalchemy.org/download.html. Then run the command 
    $ easy_install [filename]
    where [filename] is the path of the file you just downloaded. (As of this writing the name of the file you want is SQLAlchemy-0.4.0.tar.gz.)


Install any or all of these optional Python packages. For each, it is probably easiest to download the tarball and then use easy install.
  • If you are using R, install RPy (http://rpy.sourceforge.net/).  This is a Python interface for the statistical programming language R (tested with version 0.4.6).
  • If you need to use .dbf files (for example, those produced by ArcGIS), install dbfpy (http://sourceforge.net/projects/dbfpy).
  • If you are using MySQL, download MySQL-python (http://sourceforge.net/projects/mysql-python). You may need to put /usr/local/mysql/bin into your path before installing mysql-python to avoid getting an error.

You may get the following error after building and installing MySQL-python 1.2.2 on 10.3.9:
ImportError: dlcompat: dyld: /Library/Frameworks/Python.framework/Versions/2.5/Resources/Python.app/Contents/MacOS/Python can't open library: /usr/local/mysql/lib/mysql/libmysqlclient_r.15.dylib (No such file or directory, errno = 2)
This may happen because the library it is looking for is in /usr/local/mysql/lib/, and not in a subfolder called mysql. Moving the files or creating aliases should fix the problem.

Step 4: Installing Opus and UrbanSim

There are two options for getting the Opus/UrbanSim code: as a zip file, or direct from our subversion repository.  (For you modeler types, subversion in this context is the name of a version control system that nicely manages keeping track of different versions of the code; we're not talking politics.)  Particularly if you will be modifying or adding to the Opus/Urbansim code, we recommend using subversion.  This will make it easy to update to newer versions of Opus/UrbanSim and merge in any changes you've made.  You can also request any particular version of the code from subversion.  However, you do have to have a subversion client installed on your machine.

Here are instructions for each of these options.  (Well, actually, we're going to give you two ways to get Opus and UrbanSim using subversion, depending on whether you are using the Eclipse IDE.  So OK, that's 3 options in all.)

For each of these options, you construct a workspace, called say '/Users/yourusername/opusworkspace'. You'll then need to add /Users/yourusername/opusworkspace to your Python path (see Step 5 for how to do this).  The workspace will have subdirectories with the code for opus_core, urbansim, and other packages, and also a file docs with all of the documentation for that version.  The manual is in the subdirectory opus_docs/manual of the opus workspace, and the tutorials are in the tutorials subdirectory.

Option 1: Getting Opus and UrbanSim as a zip file

This is the simplest option.  Browse to our downloads page, and select a zip file for a stable release.  (Probably you want the latest stable release.)  Caution: don't get the 2006 release if you are using these directions!  That uses an older set of instructions on doing an install.  Get a 2007 release or later.

Create a workspace directory, say '/Users/yourusername/opusworkspace', and unzip the file there.

Hint: if you find yourself starting to edit the source code in any of the packages you downloaded, stop, delete the workspace directory, and get the code using subversion instead.  You'll have a much easier time later if you update your code.

Option 2: Getting Opus and UrbanSim using a command line interface to subversion

p>Suppose you want to get stable release 4.1.1. First, make sure you have a subversion client installed.  If not, you can get one from http://subversion.tigris.org/project_packages.html.   Next, in a shell (i.e. a terminal window), type the following commands:

mkdir opusworkspace
cd opusworkspace
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/4.1.1/opus_core
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/4.1.1/urbansim
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/4.1.1/eugene

The above commands check out stable release 4.1.1 -- adjust the URLs appropriately to get a different stable release. See Opus Revisions and Releases for details on this, on getting all of the packages at once instead of one by one, on checking out the development version instead of a stable release, etc.

Option 3: Getting Opus and UrbanSim using subversion and Eclipse

This is the best option if you'll be doing software development.  See Installing Opus and UrbanSim using an IDE for details.

Step 5: Defining Environment Variables 

UrbanSim requires the following environment variables.  The first tells the Python interpreter where to find the Opus/UrbanSim code. The remaining variables describe how to connect to MySQL -- leave those four out if you aren't installing MySQL.

  • PYTHONPATH - This environment variable tells Python where to look for files outside of Python's 'site-packages' directory (which is the default location for Python installers). Change PYTHONPATH to include the path to the directory containing your 'workspace' that you defined in Step 4, e.g. '/Users/yourname/opusworkspace'.  On 10.5 or later, you also need to add the diretory /Library/Frameworks/Python.framework/Versions/2.5/lib/python2.5/site-packages, as the preinstalled version of Python uses a different directory for sitepackages. Finally, make sure your directory with enthought/traits is on the PYTHONPATH.
  • MYSQLHOSTNAMEFORTESTS - This environment variable is only used for running unit tests that use MySQL. These tests use the MySQL database hostname described by this environment variable. If this environment variable is not defined, unit tests that expect this database server will not run, but will instead print a warning message.
  • MYSQLHOSTNAME - The name of the MySQL server. Use localhost for the MySQL server on the same computer. 
  • MYSQLUSERNAME - This should have the MySQL user name to be used for querying the UrbanSim database. In the lab, we use the MySQL username "urbansim" as a convenient way to allow sharing scripts among all the lab members. (Note that MySQL user names are independent of login names.) 
  • MYSQLPASSWORD - The password for the MySQL user name given above. 

On the Mac under OS X, the way to set environment variables is via an XML file '.MacOSX/environment.plist' in the user's home directory so that the environment variables are seen by the various different shells (see http://developer.apple.com/qa/qa2001/qa1067.html). Here is the sample contents of the file for these variables:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple Computer//DTD PLIST 1.0//EN"
"http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
 <dict>
 <key>DISPLAY</key>
 <string>:0.0</string>

 <key>MYSQLHOSTNAME</key>
 <string>localhost</string>

 <key>MYSQLUSERNAME</key>
 <string>urbansim</string>

 <key>MYSQLPASSWORD</key>
 <string>thesecretpassword</string>

 <key>PYTHONPATH</key>
 <string>/Users/yourusername/opusworkspace</string>

 </dict>
</plist>
