Installing Python Packages

In addition, some additional Python libraries are required, recommended, or optional for use with Opus.  You must have the required libraries to run Opus.  The recommended libraries are not necessary to run Opus, but certain functionality will be unavailable without them.  The optional libraries are Python modules that are needed only in special cases. 

Caution! There will be different binaries of these libraries for Python 2.4 and Python 2.5 - make sure you download the correct version for the Python you are using.

Required Python Packages

  • numpy 1.0.1 (or greater) (http://numpy.scipy.org/) is a fast array-processing library used throughout Opus.
  • scipy 0.5.2 (http://www.scipy.org/) is open-source software for mathematics, science, and engineering.  Opus needs the scipy.ndimage package.  
    (If you have trouble installing scipy you can install just the ndimage packages.)
  • Traits (http://code.enthought.com/traits/) is a system to define statically typed attributes for Python objects, which can then be edited via a graphical user interface via the TraitsUI package (see below).  You must install numpy and scipy first.
  • SQLAlchemy (http://www.sqlalchemy.org) allows programmatic access to a variety of database engines. Install version 0.4

Recommended Python Packages

  • If you are using MySQL, install MySQL-python 1.2.2 (http://sourceforge.net/projects/mysql-python).  This provides our connection from Python to MySQL.  (If you will build this package from scratch, you will need to install the mysql-devel package, too.)
  • PyXML 0.8.4 (http://sourceforge.net/projects/pyxml/) This Python package is needed for parsing the XML files containing SQL indicator definitions in the Opus package opus_core.  If you don?t use these you may not need this.  But if you find an import xml or import xml.dom command somewhere, then it's looking for a Python package provided by PyXML.
  • matplotlib 0.87 (http://matplotlib.sourceforge.net/) is a quite full-featured Python package for creating plots and graphs.  The binary for Windows worked fine (make sure you get the correct version for the Python that you are using, 2.4 or 2.5).  Note: version 0.90.0 does NOT work with Opus.
  • Enthought Traits (http://code.enthought.com/traits/) is a set of GUI tools designed to complement Traits and that make it very easy to create a graphical user interface to edit the traits of a Trait-based class.
    • After downloading a version of enthought.traits from Enthought, move the unzipped package to be next to site-packages directory, e.g.: 
      mv enthought.traits-1.1.0 ~/home/python24/lib/python2.4/
    • Go into the enthought/traits subdirectory of that directory, e.g.: 
      cd ~/home/python24/lib/python2.4/enthought.traits-1.1.0/enthought/traits
    • Do an "in place" build in that directory: 
      python setup_minimal.py build_ext --inplace
    • Add to your PYTHONPATH the directory containing the enthought directory, e.g.:
      ~/home/python24/lib/python2.4/enthought.traits-1.1.0

Optional Python Packages

  • If you are using R, install RPy (http://rpy.sourceforge.net/).  This is a Python interface for the statistical programming language R (tested with version 0.4.6).
  • If you need to use .dbf files (for example, those produced by ArcGIS), install dbfpy (http://sourceforge.net/projects/dbfpy).
