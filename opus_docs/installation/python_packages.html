Installing Python Packages

Some additional Python libraries are required, recommended, or optional for use with Opus.  You must have the required libraries to run Opus.  The recommended libraries are not necessary to run Opus, but certain functionality will be unavailable without them.  The optional libraries are Python modules that are needed only in special cases. 

Caution! The package you need depends on the version of python you are using. Be sure you have the right one. In most cases, the python version number will be part of the filename.

In many cases, obtaining these packages is made simpler by the use of Easy Install, a python module that can help you download and build python packages. You also need Easy Install in order to install the Traits package. For more on Easy Install, visit the PEAK webpage.

Download ez_setup.py to your computer. Run it (from the directory it's in) by typing 

$ python ez_setup.py
You can use Easy Install to download and install a package, or to install a package (either a *.egg file or a *.tar.gz) already downloaded to your computer. To install downloaded package, type the following at the command prompt: 
$ easy_install [filename]
where [filename] is the path from your current location of the downloaded file.

Required Python Packages

  • numpy 1.0.4 (or greater) (http://numpy.scipy.org/) is a fast array-processing library used throughout Opus.
  • scipy 0.7 (http://www.scipy.org/) is open-source software for mathematics, science, and engineering.  Opus needs the scipy.ndimage package.  
    (If you have trouble installing scipy you can install just the ndimage packages.)
  • Traits (http://code.enthought.com/traits/) is a system to define statically typed attributes for Python objects, which can then be edited via a graphical user interface via the TraitsUI package (see below).  You must install numpy and scipy first. The easiest way to install Traits is using Easy Install. To do this on Windows, type 
    easy_install http://code.enthought.com/enstaller/eggs/windows/xp/enthought.traits-2.0b2-py2.5-win32.egg
    at the command line. This will download and install traits for you. You may still need to add the traits directory to your python path. (For other operating systems you may need to specify a different file. Browse http://code.enthought.com/enstaller/eggs/ to find the right one.)
  • SQLAlchemy (http://www.sqlalchemy.org) allows programmatic access to a variety of database engines. Install version 0.4
  • PyXML 0.8.4 (http://sourceforge.net/projects/pyxml/) This Python package is needed for parsing XML files.
  • matplotlib 0.90.1 (http://matplotlib.sourceforge.net/) is a quite full-featured Python package for creating plots and graphs. The binary for Windows worked fine.

Recommended Python Packages

  • If you are using MySQL, install MySQL-python 1.2.2 (http://sourceforge.net/projects/mysql-python).  This provides our connection from Python to MySQL.  (If you will build this package from scratch, you will need to install the mysql-devel package, too.)
  • Enthought Traits UI (http://code.enthought.com/traits/) is a set of GUI tools designed to complement Traits and that make it very easy to create a graphical user interface to edit the traits of a Trait-based class. Browse http://code.enthought.com/enstaller/eggs/ to find the correct version and install it using easy_install.

Optional Python Packages

  • If you are using R, install RPy (http://rpy.sourceforge.net/).  This is a Python interface for the statistical programming language R (tested with version 0.4.6).
  • If you need to use .dbf files (for example, those produced by ArcGIS), install dbfpy (http://sourceforge.net/projects/dbfpy).
