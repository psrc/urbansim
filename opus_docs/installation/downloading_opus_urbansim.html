First, decide whether you want to use a stable release or the development version. For most users, the latest stable release is the best option. Pick the development version for the latest features but at the cost of code that may be less stable and that changes frequently. (See Opus Revisions and Releases for more information on version numbers.)

Getting the Sample Data and Configurations

We provide a zip file that includes sample data for Eugene and Seattle projects, template configurations for running simulations using these projects, and a source code directory. Download the zip file with sample data and configurations for either a stable release the latest development version from the Opus/UrbanSim download page. Be sure and get the zip file corresponding to your chosen version (a stable release or latest development version). Unzip the file, which will result in a directory called opus with various subdirectories. Put this directory some convenient place on your local disk. (We suggest putting it in your home directory.) See Opus Directory Tree for more information on these directories and sample data.

Getting the Source Code for a Stable Release

The zip file has a subdirectory src that already has the code for the basic packages for running Opus/UrbanSim. If you don't need any additional packages, and if you aren't modifying the code, this is all you need.

If you want to get the rest of the packages, to upgrade to a new stable release, or to switch to the development version of the code, you should check out the source code directly from our subversion repository.  (For you modeler types, subversion in this context is the name of a version control system that nicely manages keeping track of different versions of the code; we're not talking politics.)  Using subversion, it is easy to update to newer versions of Opus/UrbanSim and merge in any changes you've made.  You can also request any particular version of the code from subversion.  However, you do have to have a subversion client installed on your machine.

Before getting the code using subversion, delete all the files in the src directory that come with the zip file. (We will be replacing them with files from the repository, augmented with svn information.)

Here are two ways to get the code. The source code directory will have subdirectories with the code for opus_core, urbansim, and other packages, and also a file docs with all of the documentation for that version.  The manual is in the subdirectory opus_docs/manual of the opus workspace, and the tutorials are in the tutorials subdirectory.

These directions assume that you are getting the latest stable release. Substitute a specific version number if you want a different stable release -- the available stable releases are listed on the Opus/UrbanSim download page; or you can see a list by looking directly at the source code repository: https://trondheim.cs.washington.edu/svn/opus/tags/.

Option 1: Getting Opus and UrbanSim using a command line interface to subversion

First, make sure you have a subversion client installed (svn). If not, you can get one from http://subversion.tigris.org/project_packages.html.   Next, in a shell (i.e. a terminal window or DOS window), type the following commands. These commands assume you start the shell in whatever directory holds your opus directory.

If you have a fast connection, the simplest thing is to get all of the Opus packages at once using these commands:

cd opus
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/ src

This command checks out all of the opus packages into the src directory.

To check out a different stable release (e.g. 4.2.0), use a URL like this instead:

cd opus
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/4.2.0/ src
(latest-stable-release will be the same as the last-numbered specific version, but using latest-stable-release makes it easier to update to new releases.)

Alternatively, if you have a slow connection or limited disk space, you can check out just the minimal set of packages needed for the tutorial:

cd opus/src
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/opus_core
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/urbansim
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/urbansim_gridcell
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/eugene
svn checkout https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/opus_gui

Notice that this time we are in the src directory when checking out individual packages. The cd command above works directly for Mac and Linux. On Windows the character for subdirectories is \ instead of /, so the initial command is cd opus\src instead.

Updating to a New Stable Release

If you don't have any configurations and data of your own, the easiest course is just to delete your existing opus directory and get a fresh zip file. If you do have configurations or data and you checked out the latest stable release using the URL https://trondheim.cs.washington.edu/svn/opus/tags/latest-stable-release/, then all you need to do to update if there is a new stable release is to open a shell or DOS window, navigate to the opus/src directory that holds the source code, and use the svn update command: 

cd opus/src
svn update
If you checked out a specific numbered version for a stable release, want to switch to a different specific numbered version, or want to switch to the latest development version, you can use the switch command to svn to update your working copy of the code to a different version. For example, if you checked out all the packages and want to update to version 4.2.1, do this: 
cd opus
svn switch https://trondheim.cs.washington.edu/svn/opus/tags/4.2.1/ src

If you update the code you may need to update your data and configurations as well. The Release Notes should give information on any updates that are needed.

Getting the Source Code for the Latest Development Version

For the development version, you need to check out the source code directly from our subversion repository. (The src subdirectory of the zip file with the sample data and configurations is empty for the development version since the source code changes frequently.) Follow the above directions, except that use a different URL for the code. For example, to check out all the Opus packages at once, use: 

cd opus
svn checkout https://trondheim.cs.washington.edu/svn/opus/trunk/ src

Or if you've already checked out a stable release, here's how to switch from a stable release to the development version (again assuming you checked out all the packages):

cd opus
svn switch https://trondheim.cs.washington.edu/svn/opus/trunk/ src
